Asserts are useful in situations where you want to consider a particular condition as a failure. You can find details about available fields in each respective actions page. So, the fields for requests can be found in the [request](/floodr/benchmark-reference/actions/request), etc.

## Examples

Below are examples of how to use the feature

### Assert Status Code
For example, if you're testing redirects, and you want to assert that the response is a 301 you could setup a YAML file with:

<Code title="benchmark.yml" lang="yaml" code={`# Assume a 301 is in place for /api/latest -> /api/v3
plan:
  - name: Fetch redirected route
    request:
      url: /api/latest
    assign: latestRedirect
  - name: Assert request response code
    assert:
      key: latestRedirect.status
      value: 200 # The status is set to the LAST code (which is the 200 when /api/v3 finishes)`}/>

<Aside type="note">You **need** to do an `assign` to assert the value in later actions, the `name` will not work as a lookup</Aside>

### Assert Everything

Consider a server that returns an HTTP response of:

```http
HTTP/1.1 200 OK
Content-Length: 12
Content-Type: text/plain
Date: Sun, 05 Apr 2026 07:50:37 GMT
X-Backend: gotham
X-Request-Id: c762fb85-4e7e-4fd2-bf1e-d626f63b7a64

Hello World!
```

<Aside type="note">

If you want to follow along, below is the code for the server

<details>
    <summary>Click for code</summary>

`cargo.toml`

```toml
[package]
name = "gotham_hello"
version = "0.1.0"
edition = "2021"

[dependencies]
gotham = "0.7"
mime = "0.3"
```

`src/main.rs`

```rust
use gotham::helpers::http::response::create_response;
use gotham::hyper::{Body, Response, StatusCode};
use gotham::state::State;
use gotham::router::builder::*;
use gotham::router::Router;
use mime;

fn hello(state: State) -> (State, Response<Body>) {
    let mut response = create_response(
        &state,
        StatusCode::OK,
        mime::TEXT_PLAIN,
        "Hello World!",
    );

    response
        .headers_mut()
        .insert("x-backend", "gotham".parse().unwrap());

    (state, response)
}

fn router() -> Router {
    build_simple_router(|route| {
        route.get("/").to(hello);
    })
}

fn main() {
    println!("Listening on http://0.0.0.0:4896");
    gotham::start("0.0.0.0:4896", router());
}
```

</details>

</Aside>

We can then test the status code, presence of headers, url, and http version using:

<Code lang="yaml" title="benchmark.yml" code={`base: 'http://localhost:4896'

plan:
  # Setup request and assign it to the name gothamServer
  - name: Fetch route
    request:
      url: /
    assign: gothamServer
  # Begin Assertions
  - name: Assert request response code
    assert:
      key: gothamServer.status
      value: 200
  - name: Assert x-backend header is gotham
    assert:
      key: gothamServer.headers["x-backend"]
      value: gotham
  - name: Assert URL
    assert:
      key: gothamServer.url
      value: http://localhost:4896/
  - name: Assert http version
    assert:
      key: gothamServer.version
      value: http/1.1`}/>

### Asserting More Complex States

Assertions only allow for a simple check of equality. If you need something more dynamic, you will need to take advantage of [exec](/floodr/benchmark-reference/actions/exec) to do fancier processing. For example consider this file that checks if the status is less than 300 using python (you could also use `bash`, or some other terminal for simplicity):

<Code lang="yaml" code={`base: 'http://localhost:4896'

plan:
  - name: Fetch route
    request:
      url: /
    assign: gotham
  - name: Run python check
    exec:
      command: 'print({{gotham.status}}<300)'
    assign: codeIsValid
  - name: Assert request is valid status code
    assert:
      key: codeIsValid
      value: "True"`}/>
