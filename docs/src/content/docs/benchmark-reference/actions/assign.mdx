Floodr features an interpolation engine that will allow you to query data about responses. Assignment will expose the following fields from the response to a [`Request`](/floodr/benchmark-reference/actions/requests):

| Field | Type | Description |
|-------|------|-------------|
| status | `Number` | The HTTP status code |
| body | `Value` | The contents of the body (if they exist) | 
| headers | `Map<String, Value>` | A mapping of the available headers, for example `{request.headers["server"]}` |
| url | `String` | The retrieved url | 
| version | `String` | The HTTP version (e.g. `http/1.1`) | 

<Aside type={`note`}>The `Value` type is a mapping of any valid JSON value (i.e. `Bool`, `Number`, `String`,`Array` or `Object`) as deserialized by [`serde_json::from_str`](https://docs.rs/serde_json/latest/serde_json/fn.from_str.html). For most requests it typically coerced into a `String`, but this is not always true</Aside>

Under the hood this uses the interpolation engine to stuff values into a `floodr::actions::AssignedRequest;` struct. 

<Aside type={"danger"}>
It's important to note that actions run **sequentially**, meaning the order you define them in is relevent to their execution. This means [assignments](/floodr/benchmark-reference/assign) **must** happen before the variables are used.
</Aside>


## Examples

Below are examples of how to use the feature:

### Assert Status Code
For example, if you're testing redirects, and you want to assert that the response is a 301 you could setup a YAML file with:

<Code title="benchmark.yaml" lang="yaml" code={`# Assume a 302 is in place for /api/latest -> /api/v3
base: 'http://localhost:4896'

plan:
  - name: Fetch redirected route
    request:
      url: /api/latest
    assign: latestRedirect`}/>

`cargo.toml`

```toml
[package]
name = "gotham-redirect-example"
version = "0.1.0"
edition = "2021"

[dependencies]
gotham = "0.7"
```

`src/main.rs`

```rust
use gotham::helpers::http::response::create_empty_response;
use gotham::hyper::{Body, Response, StatusCode, Uri};
use gotham::router::builder::*;
use gotham::router::Router;
use gotham::state::State;

fn redirect_latest(state: State) -> (State, Response<Body>) {
    let response = Response::builder()
        .status(StatusCode::FOUND) // 302
        .header("Location", "/api/v3")
        .body(Body::empty())
        .unwrap();

    (state, response)
}

fn api_v3(state: State) -> (State, Response<Body>) {
    let response = Response::builder()
        .status(StatusCode::OK)
        .header("Content-Type", "text/plain; charset=utf-8")
        .body(Body::empty())
        .unwrap();

    (state, response)
}

fn router() -> Router {
    build_simple_router(|route| {
        route.get("/api/latest").to(redirect_latest);
        route.get("/api/v3").to(api_v3);
    })
}

pub fn main() {
    println!("Listening on http://127.0.0.1:4896");
    gotham::start("127.0.0.1:4896", router());
}
```

</details>

</Aside>

Now the data about the response to `/api/latest` will be assigned to the variable `latestRedirect` which can be accessed in other actions. For example in [assertions](/floodr/benchmark-reference/assert):

<Code title="benchmark.yaml" lang="yaml" code={`# Assume a 302 is in place for /api/latest -> /api/v3
base: 'http://localhost:4896'

plan:
  - name: Fetch redirected route
    request:
      url: /api/latest
    assign: latestRedirect
  - name: Assert request response code
    assert:
      key: latestRedirect.status
      value: 200
  - name: Assert new url
    assert:
      key: latestRedirect.url
      value: http://localhost:4896/api/v3`}/>
