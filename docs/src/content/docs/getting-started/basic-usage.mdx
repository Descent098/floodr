To work with floodr currently you will need to know [YAML](https://docs.ansible.com/projects/ansible/latest/reference_appendices/YAMLSyntax.html), I would recommend brushing up on it if you're not familiar. 

## Single Request

To get started let's look at a simple benchmark that just hits `http://localhost:4896/` once. Our file would look like this:

<Code title="benchmark.yml" lang="yaml" code={`base: 'http://localhost:4896' # The domain we want to hit

plan: # An array of actions to take
  - name: Fetch route
    request:
      url: /
`}/>

<Aside type="note">

If you want to follow along, below is the code for the server

<details>
    <summary>Click for code</summary>

`cargo.toml`

```toml
[package]
name = "gotham_hello"
version = "0.1.0"
edition = "2021"

[dependencies]
gotham = "0.7"
mime = "0.3"
```

`src/main.rs`

```rust
use gotham::helpers::http::response::create_response;
use gotham::hyper::{Body, Response, StatusCode};
use gotham::state::State;
use gotham::router::builder::*;
use gotham::router::Router;
use mime;

fn hello(state: State) -> (State, Response<Body>) {
    let mut response = create_response(
        &state,
        StatusCode::OK,
        mime::TEXT_PLAIN,
        "Hello World!",
    );

    response
        .headers_mut()
        .insert("x-backend", "gotham".parse().unwrap());

    (state, response)
}

fn router() -> Router {
    build_simple_router(|route| {
        route.get("/").to(hello);
    })
}

fn main() {
    println!("Listening on http://0.0.0.0:4896");
    gotham::start("0.0.0.0:4896", router());
}
```

</details>

</Aside>

To run our file we would then run:

```bash
floodr --benchmark benchmark.yml
```

## Multiple Threads & Iterations

We could now bump it up to do 100 requests across 10 threads:

<Code title="benchmark.yml" lang="yaml" code={`base: 'http://localhost:4896' # The domain we want to hit
concurrency: 10 # Number of threads to run
iterations: 100 # Number of times to run the actions defined in plan

plan: # An array of actions to take
  - name: Fetch route
    request:
      url: /`}/>

## Multiple Actions

We can also add more actions to do more things (keep in mind each action runs **per iteration**):

<Code title="benchmark.yml" lang="yaml" code={`base: 'http://localhost:4896' # The domain we want to hit
iterations: 10 # Number of times to run the actions defined in plan

plan: # An array of actions to take
  - name: Fetch route
    request:
      url: /
  - name: Fetch route
    request:
      url: /hello`}/>

This will run 20 requests (10 iterations of 1 request to `/` and 1 request to `/hello`)

<Aside type="tip">To preview which tasks will run, you can use `floodr --list-tasks`</Aside>
